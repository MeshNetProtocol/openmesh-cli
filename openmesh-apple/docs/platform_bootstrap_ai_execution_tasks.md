# OpenMesh 平台方案 AI 执行任务清单（V1）

关联主文档：`docs/platform_bootstrap_execution_master_plan.md`

## 0. 使用说明

1. 本文档用于把战略方案转换为 AI 可执行任务。  
2. 每个任务必须具备：输入、步骤、产物、验收、回滚。  
3. 每次执行前先改“状态”；执行后必须回填“产物/验收结果”。  

## 1. 任务状态定义

1. `TODO`：未开始。  
2. `IN_PROGRESS`：正在执行。  
3. `BLOCKED`：存在阻塞，需先解阻。  
4. `REVIEW`：已完成实现，待验证。  
5. `DONE`：验收通过。  
6. `SKIPPED`：本轮明确跳过并记录原因。  

## 2. 全局执行约束

1. 商用发现主路径必须是“链上直读 + 本地筛选”，不得把 Cloudflare 作为必需依赖。  
2. 冷启动主路径必须是“官方临时受限引导 -> 社区私人配置接力”。  
3. 新老用户必须分层体验，不得用单一路径覆盖全部用户。  
4. 钱包只在商用关键动作触发，不得阻塞新用户首连。
