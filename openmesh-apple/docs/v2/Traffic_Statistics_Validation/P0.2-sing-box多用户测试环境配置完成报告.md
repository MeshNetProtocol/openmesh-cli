### 2.4 管理脚本

创建了 4 个可执行脚本：

**[stop-nodes.sh](test-environment/stop-nodes.sh)**

- 支持参数切换连接节点：`./start-client.sh node-b`
- 自动修改配置文件中的端口

**[test-traffic.sh](test-environment/test-traffic.sh)**

## 4. 技术特性

### 4.1 单机测试支持

- ✅ 客户端和服务端在同一台 Mac 上运行
- ✅ 通过网卡绑定避免流量死循环
- ✅ 所有节点监听 127.0.0.1

### 4.2 双机部署就绪

配置保持灵活性，迁移到双机只需：
1. 修改服务端 `listen: "127.0.0.1"` → `"0.0.0.0"`
2. 修改客户端 `server: "127.0.0.1"` → 服务器 IP
3. 移除 `bind_interface: "en0"`（不再需要）

### 4.3 多用户支持

- ✅ 3 个测试用户：alice, bob, charlie
- ✅ 每个用户独立密码
- ✅ 支持多用户并发连接

### 4.4 节点切换

- ✅ 客户端可以轻松切换连接不同节点
- ✅ 通过脚本参数或修改配置文件
- ✅ 所有节点配置一致，便于测试

## 8. 风险与注意事项

### 8.1 已知限制

- ⚠️ **managed 模式限制**: `managed: true` 不支持在配置文件中静态定义用户，需要通过 SSM API 动态管理
  - **当前方案**: 使用静态用户配置（移除 `managed: true`）
  - **P0.3 计划**: 实现 SSM API 服务，支持动态用户管理和流量统计
- ⚠️ 暂未实现流量统计上报（将在 P0.4 实现）
- ⚠️ 单机测试需要网卡绑定，macOS 可能需要授权

### 8.2 安全注意

- ⚠️ `passwords.txt` 包含敏感信息，不应提交到 git
- ⚠️ 当前配置仅用于测试，生产环境需要更强的密码

### 8.3 macOS 特定

- macOS 可能提示允许 sing-box 网络访问
- 需要在"系统设置 → 隐私与安全性"中允许
