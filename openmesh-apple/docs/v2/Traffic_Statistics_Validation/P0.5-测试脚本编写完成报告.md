## 3. 测试脚本详解

### 3.1 流量注入脚本

**文件**：`test-traffic-injection.sh`

**功能**：
- 通过 SOCKS5 代理下载测试文件
- 产生真实的代理流量
- 支持 1MB/10MB/50MB 三种文件大小
- 记录下载时间和速度

**用法**：
```bash
./test-traffic-injection.sh 10  # 下载 10MB 文件
```

**核心逻辑**：
```bash
curl -x socks5h://127.0.0.1:1080 \
    -o /tmp/downloaded-file \
    http://127.0.0.1:8888/test-10mb.bin
```

### 3.2 流量验证脚本

**文件**：`test-traffic-verification.sh`

**功能**：
- 产生已知大小的流量（10MB）
- 查询记账服务的统计数据
- 对比实际流量和统计流量
- 计算误差百分比

**验证标准**：
- 流量统计误差 < 5%

**测试流程**：
1. 查询初始流量统计
2. 下载 10MB 文件
3. 等待 15 秒（流量上报）
4. 查询最终流量统计
5. 计算增量和误差

### 3.3 节点切换测试脚本

**文件**：`test-node-switching.sh`

**功能**：
- 用户在不同节点间切换
- 验证流量在所有节点正确汇总
- 验证流量桶统一管理

**测试场景**：
- alice 在 node-a 下载 5MB
- alice 切换到 node-b 下载 5MB
- alice 切换到 node-c 下载 5MB
- 验证 alice 总流量为 15MB

**验证标准**：
- 多节点流量汇总误差 < 10%

### 3.4 流量桶耗尽测试脚本

**文件**：`test-quota-exhaustion.sh`

**功能**：
- 创建小配额用户（5MB）
- 持续上报流量直到耗尽
- 验证流量桶耗尽后返回 "block" 状态

**测试流程**：
1. 创建测试用户（5MB 配额）
2. 模拟流量上报（每次 2MB）
3. 持续上报直到收到 "insufficient" 状态
4. 验证最终剩余流量 < 1MB
