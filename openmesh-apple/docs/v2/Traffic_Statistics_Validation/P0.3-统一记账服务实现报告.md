## 3. 数据库设计

### 3.1 users 表

存储用户流量桶信息：

```sql
CREATE TABLE users (
    user_id TEXT PRIMARY KEY,
    provider_id TEXT DEFAULT 'provider_a',
    total_quota INTEGER NOT NULL,      -- 总配额（字节）
    used_upload INTEGER DEFAULT 0,     -- 已用上传（字节）
    used_download INTEGER DEFAULT 0,   -- 已用下载（字节）
    remaining INTEGER NOT NULL,        -- 剩余（字节）
    price_rate REAL DEFAULT 100.0,     -- 价格比率：1 USDC = 100MB
    purchased_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 traffic_reports 表

存储节点上报的流量记录：

```sql
CREATE TABLE traffic_reports (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    node_id TEXT NOT NULL,
    user_id TEXT NOT NULL,
    upload_bytes INTEGER NOT NULL,
    download_bytes INTEGER NOT NULL,
    reported_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);
```

## 5. 核心功能实现

### 5.1 流量汇总逻辑

```python
def report_traffic(node_id, user_id, upload_bytes, download_bytes):
    with transaction():
        # 1. 查询用户流量桶
        user = get_user(user_id)

        # 2. 检查流量桶是否足够
        total_bytes = upload_bytes + download_bytes
        if user.remaining < total_bytes:
            return {
                "status": "insufficient",
                "remaining": user.remaining,
                "action": "block"
            }

        # 3. 原子性扣减流量桶
        user.used_upload += upload_bytes
        user.used_download += download_bytes
        user.remaining -= total_bytes
        user.updated_at = now()
        user.save()

        # 4. 记录流量上报
        report = TrafficReport(
            node_id=node_id,
            user_id=user_id,
            upload_bytes=upload_bytes,
            download_bytes=download_bytes
        )
        report.save()

        return {
            "status": "ok",
            "remaining": user.remaining,
            "action": "continue"
        }
```

### 5.2 流量桶管理

- **充值**：创建用户时根据 USDC 金额和价格比率计算流量配额
- **扣减**：每次上报时原子性扣减
- **查询**：实时查询剩余流量
