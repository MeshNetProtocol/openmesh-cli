# 计费模型变更说明

> 创建时间：2026-03-22
> 状态：已确认

## 1. 变更概述

OpenMesh V2 的计费模型从**实时计费 + 双向签名**改为**预付费 + 中心化扣减**。

---

## 2. 旧模型（已废弃）

### 2.1 核心流程

```
用户使用流量 → 客户端统计 → 服务端统计 → 双向签名 → 生成凭条 → 互相备份 → 实时计费
```

### 2.2 关键特性

- **实时计费**: 按实际使用流量实时计费
- **双向签名**: 客户端和服务端都签名确认流量使用
- **凭条备份**: PaymentReceipt 和 UsageReceipt 双方互相备份
- **凭条恢复**: 任一方丢失可向对方恢复
- **x402 角色**: 用于实时流量计费签名
- **流量桶**: 免费 40MB，续费 40MB，剩余 5MB 时触发续费

### 2.3 问题

- ❌ 复杂度高：需要双向签名、凭条备份、恢复机制
- ❌ 实时性要求高：流量统计和计费必须实时同步
- ❌ 状态管理复杂：需要维护双方的凭条状态
- ❌ 开发成本高：客户端和服务端都需要复杂的签名和存储逻辑

---

## 3. 新模型（当前）

### 3.1 核心流程

```
用户预付 USDC → x402 支付 → 兑换流量配额 → 使用流量 → 中心化扣减 → 配额耗尽 → 阻断
```

### 3.2 关键特性

- **预付费**: 用户通过 x402 向服务商预付 USDC
- **直接兑换**: USDC 直接兑换成流量配额（如 1 USDC = 100MB）
- **价格差异**: 不同服务商独立定价（如服务商 A: 100MB/USDC，服务商 B: 80MB/USDC）
- **永久有效**: 流量配额永久有效，直到用完
- **中心化扣减**: 记账服务中心化扣减流量配额，无需双向签名
- **无凭条**: 不需要 PaymentReceipt 和 UsageReceipt
- **服务商独立**: 每个服务商独立运营记账服务，互不感知
- **x402 角色**: 只用于预付费支付，不再用于实时计费签名

### 3.3 优势

- ✅ 简单直接：预付费模型，用户易于理解
- ✅ 开发成本低：无需双向签名、凭条备份、恢复机制
- ✅ 实时性要求低：流量统计可以定期上报（如每 10 秒）
- ✅ 状态管理简单：只需维护流量配额余额
- ✅ 服务商自主：每个服务商独立定价和运营
- ✅ 平台开源：平台提供开源记账服务代码，服务商可自行部署

---

## 4. 对比表

| 维度 | 旧模型（实时计费） | 新模型（预付费） |
|------|------------------|-----------------|
| **支付时机** | 实时按流量计费 | 预付费购买流量包 |
| **x402 角色** | 实时流量计费签名 | 预付费支付 USDC |
| **流量有效期** | 按桶消耗（40MB/桶） | 永久有效，直到用完 |
| **签名机制** | 双向签名（客户端 + 服务端） | 单向签名（x402 支付签名） |
| **凭条管理** | PaymentReceipt + UsageReceipt | 无需凭条 |
| **凭条备份** | 双方互相备份 | 无需备份 |
| **凭条恢复** | 任一方丢失可恢复 | 无需恢复 |
| **流量统计** | 客户端 + 服务端双向统计 | 服务端统计 |
| **流量扣减** | 实时扣减 | 中心化扣减 |
| **价格模型** | 统一价格（40MB/桶） | 服务商独立定价 |
| **服务商关系** | 平台统一管理 | 服务商独立运营 |
| **记账服务** | 平台提供 | 服务商自行部署（平台提供开源代码） |
| **开发复杂度** | 高 | 低 |
| **用户体验** | 复杂（需要理解桶、续费等概念） | 简单（预付费，类似充值话费） |

## 5. 架构变化

### 5.1 旧架构（已废弃）

```
客户端                          服务端
  │                              │
  ├─ 流量统计                    ├─ 流量统计
  ├─ 签名                        ├─ 签名
  ├─ PaymentReceipt 存储         ├─ PaymentReceipt 存储
  ├─ UsageReceipt 存储           ├─ UsageReceipt 存储
  ├─ 凭条恢复                    ├─ 凭条恢复
  └─ x402 实时计费签名           └─ x402 实时计费验证
```

### 5.2 新架构（当前）

```
客户端                          服务商
  │                              │
  ├─ x402 预付费支付             ├─ Payment Service（验证支付）
  ├─ 查询流量配额                ├─ Metering Service（流量汇总）
  └─ 使用流量                    ├─ 中心化扣减流量配额
                                 └─ 配额不足时阻断
```

## 6. 数据模型变化

### 6.1 旧模型数据结构（已废弃）

```json
// PaymentReceipt
{
  "user_id": "user_0x1234abcd",
  "provider_id": "provider_a",
  "amount": 0.01,
  "bucket_size": 40000000,
  "user_signature": "0x...",
  "provider_signature": "0x...",
  "timestamp": "2026-03-22T10:00:00Z"
}

// UsageReceipt
{
  "user_id": "user_0x1234abcd",
  "provider_id": "provider_a",
  "upload_bytes": 5000000,
  "download_bytes": 15000000,
  "user_signature": "0x...",
  "provider_signature": "0x...",
  "timestamp": "2026-03-22T10:30:00Z"
}
```

### 6.2 新模型数据结构（当前）

```json
// 流量配额
{
  "user_id": "user_0x1234abcd",
  "provider_id": "provider_a",
  "total_quota": 500000000,      // 总配额（字节）
  "used_upload": 50000000,       // 已用上传（字节）
  "used_download": 150000000,    // 已用下载（字节）
  "remaining": 300000000,        // 剩余（字节）
  "price_rate": 100,             // 1 USDC = 100MB
  "purchased_at": "2026-03-22T10:00:00Z",
  "updated_at": "2026-03-22T14:30:00Z",
  "expires_at": null             // 永久有效
}

// 支付记录（可选，用于对账）
{
  "user_id": "user_0x1234abcd",
  "provider_id": "provider_a",
  "usdc_amount": 5.0,
  "bandwidth_purchased": 500000000,
  "payment_signature": "0x...",
  "tx_hash": "0x...",
  "timestamp": "2026-03-22T10:00:00Z"
}
```

## 7. API 变化

### 7.1 旧 API（已废弃）

```bash
# 续费流量桶
POST /api/v1/bucket/renew
{
  "user_id": "user_0x1234abcd",
  "payment_signature": "0x...",
  "amount": 0.01
}

# 查询凭条
GET /api/v1/receipts/{user_id}

# 恢复凭条
POST /api/v1/receipts/recover
```

### 7.2 新 API（当前）

```bash
# 预付费购买流量
POST /api/v1/users/{user_id}/purchase
{
  "usdc_amount": 5.0,
  "payment_signature": "0x...",
  "tx_hash": "0x..."
}

# 查询流量配额
GET /api/v1/users/{user_id}/quota

# 流量上报（节点调用）
POST /api/v1/metering/report
{
  "node_id": "node_a",
  "user_id": "user_0x1234abcd",
  "upload_bytes": 1000000,
  "download_bytes": 5000000
}
```

## 9. 迁移建议

对于已经开始开发的部分：

1. **停止开发**：
   - PaymentReceipt 和 UsageReceipt 相关代码
   - 双向签名机制
   - 凭条备份和恢复逻辑

2. **重点开发**：
   - 预付费购买流量接口
   - 流量配额管理
   - 中心化扣减逻辑
   - 服务商价格配置

3. **保留开发**：
   - 流量统计和上报
   - x402 支付验证（但角色变为预付费支付）
   - 用户管理
