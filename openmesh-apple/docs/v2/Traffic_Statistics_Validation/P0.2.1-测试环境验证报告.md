## 5. 关键发现

### 5.1 配置问题与解决

**问题 1**: `managed: true` 模式冲突
- **现象**: 节点启动失败，错误信息 "users and destinations options are not supported in managed servers"
- **原因**: `managed: true` 模式不允许在配置文件中静态定义用户
- **解决**: 移除 `managed: true`，使用静态用户配置
- **影响**: 暂时无法使用 SSM API 动态管理用户，将在 P0.3 实现

**问题 2**: 端口占用
- **现象**: 客户端启动失败，端口 1080 被占用
- **原因**: 之前的客户端进程未正常退出
- **解决**: 手动 kill 旧进程
- **建议**: 后续可以改进 start-client.sh 脚本，自动检测并清理旧进程

### 5.2 日志配置

- 日志级别设置为 `warn`，只记录警告和错误
- 测试期间所有节点日志为空，说明无任何警告或错误
- 连接顺畅，无异常情况
