---

## 2. 测试脚本清单

### 2.1 流量注入脚本

**文件**：`test-traffic-injection.sh`

**功能**：
- 通过 SOCKS5 代理产生真实流量
- 下载/上传指定大小的文件
- 支持指定用户和节点
- 记录实际产生的流量

**测试场景**：
- 下载 1MB 文件
- 下载 10MB 文件
- 下载 50MB 文件
- 上传测试（如果支持）

### 2.2 流量验证脚本

**文件**：`test-traffic-verification.sh`

**功能**：
- 产生已知大小的流量
- 查询记账服务的统计数据
- 对比实际流量和统计流量
- 计算误差百分比

**验证指标**：
- 流量统计准确性（误差 < 5%）
- 流量桶扣减正确性
- 上报延迟（< 15秒）

### 2.3 多用户并发测试脚本

**文件**：`test-concurrent-users.sh`

**功能**：
- 同时启动多个客户端（不同用户）
- 并发产生流量
- 验证流量统计不冲突
- 验证流量桶独立扣减

**测试场景**：
- 3 个用户同时下载文件
- 验证每个用户的流量独立统计

### 2.4 节点切换测试脚本

**文件**：`test-node-switching.sh`

**功能**：
- 用户在不同节点间切换
- 验证流量在所有节点正确汇总
- 验证流量桶统一管理

**测试场景**：
- alice 在 node-a 下载 5MB
- alice 切换到 node-b 下载 5MB
- 验证 alice 总流量为 10MB

### 2.5 流量桶耗尽测试脚本

**文件**：`test-quota-exhaustion.sh`

**功能**：
- 创建小配额用户
- 持续产生流量直到耗尽
- 验证流量桶耗尽后的行为
- 验证充值后恢复正常

**测试场景**：
- 创建 10MB 配额的测试用户
- 下载文件直到配额耗尽
- 验证记账服务返回 "block" 状态
- 充值后验证恢复

## 3. 测试数据准备

### 3.1 测试文件服务器

需要一个可以下载测试文件的服务器：
- 选项 A：使用公共测试文件（如 speed.cloudflare.com）
- 选项 B：本地启动简单的 HTTP 服务器

**推荐**：使用 Python 启动本地 HTTP 服务器
```bash
python3 -m http.server 8888
```

### 3.2 测试文件

创建不同大小的测试文件：
```bash
dd if=/dev/zero of=test-1mb.bin bs=1M count=1
dd if=/dev/zero of=test-10mb.bin bs=1M count=10
dd if=/dev/zero of=test-50mb.bin bs=1M count=50
```

## 4. 测试流程

### 4.1 基础流量注入测试

```bash
# 1. 启动所有服务
./start-nodes.sh
./start-metering.sh
./start-collectors.sh

# 2. 启动客户端（alice 连接 node-a）
./start-client.sh

# 3. 注入流量
./test-traffic-injection.sh 10  # 下载 10MB

# 4. 验证流量统计
./test-traffic-verification.sh
```

### 4.2 多用户并发测试

```bash
# 1. 启动 3 个客户端（不同用户，不同端口）
./start-client.sh alice node-a 1080 &
./start-client.sh bob node-b 1081 &
./start-client.sh charlie node-c 1082 &

# 2. 并发注入流量
./test-concurrent-users.sh

# 3. 验证每个用户的流量独立统计
```

### 4.3 节点切换测试

```bash
# 1. alice 在 node-a 下载 5MB
./start-client.sh alice node-a
./test-traffic-injection.sh 5

# 2. alice 切换到 node-b 下载 5MB
./start-client.sh alice node-b
./test-traffic-injection.sh 5

# 3. 验证 alice 总流量为 10MB
./test-node-switching.sh
```
