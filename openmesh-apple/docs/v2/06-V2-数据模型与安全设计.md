# 06-V2-数据模型与安全设计

## 1. 文档目的

本文档定义：

- 钱包安全边界
- signer session 设计
- 支付收据和用量凭条的数据模型
- 商用供应商热路径的安全要求

---

## 2. 钱包安全原则

1. 助记词不得明文持久化
2. 私钥不得明文持久化
3. 私钥明文只允许在 signer session 中存在于内存
4. 预付费支付必须有上限
5. 钱包易用性和安全必须同时成立

---

## 3. 钱包落盘方案

推荐方案：

### 3.1 加密 blob

允许把钱包材料以加密 blob 形式存储在本地。  
这个 blob 不可直接使用，必须依赖设备绑定能力解密。

推荐实现：

1. 先生成随机 `walletDataKey`
2. 用 `walletDataKey` 加密钱包材料形成 blob
3. 再用设备绑定密钥包裹 `walletDataKey`
4. 设备绑定密钥放入 Keychain / Secure Enclave 体系

### 3.2 设备绑定保护

优先使用：

- Keychain
- Secure Enclave
- LocalAuthentication / Touch ID / 设备凭证

### 3.3 解锁流程

1. 用户首次创建或恢复钱包
2. 钱包材料被加密后落盘
3. 用户在需要自动支付前解锁一次
4. 系统通过设备绑定能力解开 `walletDataKey`
5. 再解开钱包材料
6. 解锁后只在内存中持有可签名私钥

### 3.4 快捷解锁

若用户启用本机生物识别或设备认证，可允许：

- Touch ID
- 系统认证
- 本机安全凭证

用于重新打开 signer session。  
但无论采用何种快捷解锁方式，助记词和私钥都不得明文落盘。

---

## 4. signer session 设计

### 4.1 目标

避免：

- 每次购买流量都重新输入口令

同时避免：

- 私钥长时间明文常驻

### 4.2 建议字段

- `unlockedAt`
- `unlockExpiresAt`
- `singlePaymentLimit` (单次支付上限)
- `dailyPaymentLimit` (每日支付上限)
- `dailySpent` (今日已支付)
- `requiresManualConfirm`
- `unlockMethod`
- `deviceBindingEnabled`

### 4.3 自动锁定条件

- 超时
- App 退出
- 用户手动锁定
- 每日额度耗尽
- 设备锁屏或系统安全上下文失效

### 4.4 风险控制

若钱包已解锁但支付超限：

- 不得自动签名
- 必须要求用户重新确认

## 5. 数据模型

### 5.1 TrafficQuota（流量配额）

职责：

- 记录用户在某个服务商的流量配额
- 追踪流量使用情况

保存位置：

- 服务商记账服务数据库

建议字段：

- `quotaId`
- `providerId`
- `userAddress`
- `totalQuota` (总配额，字节)
- `usedUpload` (已用上传，字节)
- `usedDownload` (已用下载，字节)
- `remaining` (剩余，字节)
- `priceRate` (购买时的价格比率，如 100 表示 1 USDC = 100MB)
- `purchasedAt`
- `updatedAt`
- `expiresAt` (null = 永久有效)

### 5.2 PurchaseRecord（购买记录）

职责：

- 记录用户的预付费购买历史
- 用于对账和审计

保存位置：

- 服务商记账服务数据库

建议字段：

- `recordId`
- `providerId`
- `userAddress`
- `usdcAmount` (支付的 USDC 数量)
- `bandwidthPurchased` (购买的流量，字节)
- `priceRate` (购买时的价格比率)
- `paymentSignature` (x402 支付签名)
- `txHash` (链上交易哈希，可选)
- `purchasedAt`

---

## 6. 预付费购买流程安全要求

1. x402 支付签名必须验证
2. 支付金额必须与购买的流量配额匹配
3. 流量配额必须原子性增加
4. 购买记录必须持久化

默认建议：

- 最小购买金额：0.1 USDC
- 最大单次购买金额：100 USDC
- 价格比率由服务商独立设置（如 1 USDC = 100MB）

---

## 7. 数据平面安全要求

1. 地址或地址派生值必须成为可统计的用户标识
2. 数据平面必须支持按地址或 session 停供
3. 节点原始统计要能汇总到地址级视图

---

## 8. 官方临时受限访问的安全边界

官方临时受限访问允许继续简化：

- 可不要求钱包
- 可不要求完整收据体系
- 可只做基础限额与日志

但不得污染商用场景的正式设计。
