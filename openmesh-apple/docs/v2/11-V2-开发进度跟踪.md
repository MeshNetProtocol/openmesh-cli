## 阶段 1：导航与 OpenMesh UI 骨架 ✅

**完成时间**：本会话

### 核心设计说明

本 App 是 **菜单栏弹窗（MenuBarExtra）** 架构，无独立主窗口侧边栏。
Tab 切换在 `OpenMeshMacApp.swift` 的 `MenuBarTab` 枚举中定义，顶部 Tab 栏为 `MenuTopTabBar`。
OpenMesh 插入位置：**Dashboard → OpenMesh → Market → Settings**。

### 修改文件

| 文件 | 改动内容 |
|------|----------|
| `OpenMeshMacApp.swift` | `MenuBarTab` 枚举新增 `.openMesh = "OpenMesh"`，插入 `.settings` 和 `.trafficMarket` 之间；`MenuBarWindowContent` switch 新增 `.openMesh` 分支渲染 `OpenMeshView()`；`MenuTopTabBar` 改为 `ForEach(MenuBarTab.allCases)` 等宽排列，支持 4 个 tab |
| `NavigationPage.swift` | 恢复原样（无改动，侧边栏与本次需求无关） |

### 新增文件

| 文件 | 路径 | 说明 |
|------|------|------|
| `OpenMeshView.swift` | `views/openmesh/` | 主容器：背景 + 顶部状态条 + 分段切换 + 内容区 |
| `OpenMeshSegmentControl.swift` | `views/openmesh/` | 分段切换控件："链上供应商" / "DIY 自建" |
| `OpenMeshTopBarView.swift` | `views/openmesh/` | 顶部钱包状态条（阶段 1 静态占位：无账户状态） |
| `OpenMeshSupplierListView.swift` | `views/openmesh/` | 供应商列表（阶段 1 含 3 条 mock 卡片） |
| `OpenMeshDIYView.swift` | `views/openmesh/` | DIY 引导页：完全独立，不依赖钱包/链上 |

### 已知问题与修复记录

| 问题 | 原因 | 修复 |
|------|------|------|
| 误将 OpenMesh 插入侧边栏 `NavigationPage` | 误读 UI 架构，本 App 是 MenuBarExtra 无侧边栏 | 恢复 `NavigationPage.swift`，改为修改 `OpenMeshMacApp.swift` 的 `MenuBarTab` |
| `@Previewable @State` 编译错误 | 需 Xcode 16+，低版本不支持 | 改为私有 `PreviewWrapper` struct |

### 阶段 1 各文件占位说明（后续替换计划）

| 文件 | 当前状态 | 替换阶段 |
|------|----------|----------|
| `OpenMeshTopBarView` | 静态"无账户"占位 | 阶段 3：接入 `MeshWalletStore` |
| `OpenMeshSupplierListView` | 3 条 mock 卡片 | 阶段 4：接入 `OnChainSupplierViewModel` |
| `OpenMeshDIYView` 按钮 | no-op 占位 | 阶段 5：接入教程链接 + `OfflineImportWindowManager` |
