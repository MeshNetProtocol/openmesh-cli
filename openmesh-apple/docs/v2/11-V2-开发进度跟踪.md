# 11-V2-开发进度跟踪

> 更新时间：2026-03-12

## 1. 当前状态

已完成：

- V2 文档已根据最新产品思想重构
- 官方临时受限访问、自建节点、商用供应商、私人节点四种场景已重新划分
- 商用供应商正式架构已改为：
  - 链上发现
  - x402 小额自动支付
  - 流量桶预续费
  - 供应商签名凭条
  - 去中心化收款

未开始：

- 钱包 signer session 实现
- 链上 registry 读取实现
- 商用供应商参考服务端
- x402 流量桶热路径
- PaymentReceipt / UsageReceipt 存储与恢复

## 2. 下一步建议

1. 先把消息模型和 API 结构补成可开发级别
2. 再实现钱包 signer session
3. 再实现商用供应商参考服务端
4. 最后打通客户端切换和自动支付闭环

## 3. 当前最重要的事实

当前的方向已经不再是“在现有 VPN 客户端里加一点区块链功能”，而是：

**建立一个以链上供应商发现、x402 自动支付和供应商签名凭条为核心的 OpenMesh 商用架构。**
