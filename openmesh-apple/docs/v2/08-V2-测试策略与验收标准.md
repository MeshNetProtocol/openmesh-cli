# 08-V2-测试策略与验收标准

## 1. 文档目的

本文档定义新版 V2 的测试重点和验收门槛。

---

## 2. 核心测试目标

1. 官方临时受限访问可继续工作
2. 私人节点和 DIY 仍独立可用
3. 商用供应商可从链上发现
4. 自动支付热路径可用
5. 预续费阈值逻辑正确
6. 凭条恢复机制成立
7. 钱包安全和自动支付限制有效

---

## 3. 商用供应商主流程测试

### 3.1 首次商用接入

必须验证：

- 用户已有钱包和 USDC
- 能发现供应商
- 能获取 metadata
- 能安装并使用商用配置

### 3.2 免费桶

必须验证：

- 首个免费桶可用
- 免费桶消耗正确

### 3.3 预续费

必须验证：

- 桶剩余量低于阈值时触发续费
- 不是等到完全耗尽才发起 `402`

### 3.4 自动支付

必须验证：

- 已解锁 signer session 时可自动支付
- 超出单次上限时不自动支付
- 超出每日上限时不自动支付

### 3.5 放行

必须验证：

- x402 支付通过后立即放行下一桶
- 不依赖链上余额轮询才能继续使用

### 3.6 凭条

必须验证：

- 支付后生成 PaymentReceipt
- 会话结束后生成 UsageReceipt
- 客户端可恢复服务端丢失的最新凭条
- 服务端可恢复客户端丢失的最新凭条

---

## 4. 钱包安全测试

必须验证：

- 助记词不明文落盘
- 私钥不明文落盘
- signer session 结束后内存中无可用私钥
- 设备绑定解锁有效

---

## 5. 发布阻断条件

若满足以下任一条件，不允许进入稳定发布：

- 官方临时受限访问不可用
- 商用供应商仍然依赖中心化账本
- 仍然在桶耗尽后才发起续费
- 自动支付无单次或每日限制
- 支付后仍需等待链上轮询才能继续供给
- 凭条不可恢复
