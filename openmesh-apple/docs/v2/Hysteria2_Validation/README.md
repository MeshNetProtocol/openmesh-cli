# Hysteria2 技术验证

> **目录目的**：存放 Phase 0 的 Hysteria2 技术验证相关文档和脚本。

---

## 验证目标

验证 Hysteria2 的核心能力：
1. 用户级流量统计准确性
2. 多节点流量汇总可行性
3. Traffic Stats API 可用性
4. 性能指标达标

---

## 验证计划

详见主文档：[01-核心需求与技术选型.md](../01-核心需求与技术选型.md#5-关键技术能力验证)

---

## 验证内容

### 1. 环境搭建

- [ ] 安装 Hysteria2 服务端
- [ ] 配置多用户认证
- [ ] 启用 Traffic Stats API
- [ ] 配置 sing-box 客户端

### 2. 单节点验证

- [ ] 配置 2 个测试用户
- [ ] 验证用户级流量统计
- [ ] 测试 Traffic Stats API 准确性
- [ ] 对比实际流量和统计流量

### 3. 多节点验证

- [ ] 部署 3 个 Hysteria2 节点
- [ ] 实现简单的流量汇总服务
- [ ] 验证多节点流量汇总准确性
- [ ] 测试节点切换场景

---

## 验收标准

| 验证项 | 验收标准 |
|--------|---------|
| 多用户配置 | ✅ 支持 10+ 用户 |
| Traffic Stats API | ✅ 返回用户级数据 |
| 流量统计准确度 | ✅ 误差 < 1% |
| 用户隔离 | ✅ 流量互不干扰 |
| 流量汇总准确性 | ✅ 无重复计数 |
| 节点切换 | ✅ 流量连续统计 |
| 节点故障 | ✅ 自动恢复 |
