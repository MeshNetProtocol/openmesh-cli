# MeshFlux V2 升级文档总览

> 本目录用于组织 MeshFlux macOS 客户端 V2 升级的完整工程文档。  
> 本次升级的核心目标是在不破坏 V1 既有能力的前提下，引入 OpenMesh 入口、链上供应商浏览、内嵌钱包、x402 支付以及 DIY 自建引导能力，并形成一套可评审、可实施、可测试、可验收、可回滚的工程文档体系。

---

## ⚡ 核心产品定位（必读，所有文档的理解前提）

**MeshFlux V2 要做的是：VPN 服务提供商界的 OpenRouter。**

OpenRouter 是大语言模型领域的统一 API 层——它不自建模型，而是聚合所有主流模型提供商，让用户在一个界面内比较、切换和使用不同的模型，按用量计费（$/百万 tokens）。

MeshFlux V2 在 VPN 领域做同样的事：

| OpenRouter 概念 | MeshFlux V2 对应 |
|---|---|
| 模型提供商（OpenAI、Anthropic…） | VPN 供应商（链上注册的节点服务商） |
| 当前使用的模型 | 当前激活的供应商 |
| 按百万 tokens 计费 | 按 MB 流量计费（金豆/MB） |
| 托管模型（直接用，无需部署） | 商用供应商（自动配置安装，无需手动） |
| 自定义 API 端点（自托管模型） | DIY 自建（用户自己的节点，手动导入） |
| 切换模型 | 切换供应商 |

### 这个定位对产品设计的直接影响

1. **界面主体是「当前在用谁」**，而不是「给你一个安装列表」。OpenRouter 打开就显示你在用哪个 model，MeshFlux V2 的 OpenMesh 页面顶部同理：主体是当前激活供应商的状态卡片。

2. **用户的核心动作是「切换」**，而不是「安装」。供应商之间的差异是价格（金豆/MB）、服务类型（通用/AI 专用/电商专用）、地区覆盖和节点数。用户像选模型一样选供应商。

3. **商用供应商的配置安装是自动的**，用户不感知。就像 OpenRouter 上调用 GPT-4 不需要你自己部署一样，选中商用供应商后，客户端自动完成配置拉取和安装，首次会展示进度界面，但这是背景过程而非主操作。

4. **DIY 模式是「自定义 API 端点」的类比**。对于有自己服务器的用户，相当于在 OpenRouter 里填入自己的模型 API——客户端不托管、不管理，用户完全自己控制，手动导入配置。

5. **价格展示的心智模型对齐 token 定价**。「0.05 金豆/MB」对有 AI 使用经验的用户来说是直觉友好的，和「$0.03/百万 tokens」是同一套认知框架。

### AI 和开发者的使用要求

**在阅读任何其他文档、设计任何界面、编写任何代码之前，必须先内化以上定位。**  
任何时候设计出现以下信号，说明已经偏离这个定位：
- 界面主体变成了「供应商安装列表」→ 跑偏了，应该是「当前在用谁 + 可切换列表」
- 用户动作描述是「安装」而非「切换」→ 跑偏了
- 价格单位不是「金豆/MB」→ 跑偏了
- DIY 路径依赖钱包或链上状态→ 跑偏了，DIY 是完全独立的自定义端点通道

## 7. 关键术语

### OpenMesh
V2 新增的客户端主入口，用于承载链上供应商浏览、钱包、支付与 DIY 引导能力。

### 金豆
面向用户展示的余额与支付单位，业务上等价映射为 Base 链上的 USDC，界面不直接展示链上技术词汇。

### 金豆账户
面向用户的账户抽象，底层对应一个轻量 EVM 钱包地址，用于余额展示、支付签名和充值接收。

### 链上供应商
通过协议注册、可被客户端读取与安装配置的供应商实体。

### DIY 自建
用户自行搭建和导入配置的使用模式，不依赖商业供应商购买流程。

### x402 支付
用于在客户端拉取配置前完成授权支付的协议流程，是 V2 支付链路的关键能力之一。

## 10. 质量与发布门槛

V2 在进入发布或评审前，必须同时满足：

- 自动化测试全部通过
- blockchain 目录覆盖率达到要求
- 无真实网络测试副作用
- 无真实 Keychain 测试副作用
- 钱包与支付安全边界成立
- OpenMesh 不破坏现有 V1 主流程
- 出现问题时具备入口隐藏、能力关闭或阶段回退方案

详见：

- `08-V2-测试策略与验收标准.md`

## 11. AI 协作开发注意事项

AI 在参与本项目时，必须遵守以下原则：

- 先读真实代码，再给方案
- 先读执行约束，再改代码
- 新增优先，最小修改旧代码
- 不改现有 VPN 核心逻辑
- 不改现有市场核心逻辑
- UI 不直接做网络、合约、Keychain 或签名
- 私钥只能进 Keychain，助记词不能落盘
- 模块完成即补测试
- 发现文档与代码冲突时，必须明确指出

详见：

- `10-AI-执行约束与代码一致性规则.md`
