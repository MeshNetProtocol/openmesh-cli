> **核心定位**：OpenMesh 要成为 VPN 行业的 OpenRouter，实现供应商快速切换和去中心化结算。

## 文档概览

本文档集定义 MeshFluxMac 客户端的 V2 升级方案，基于 **Hysteria2 协议**实现多用户流量统计和预付费配额管理。

### 核心技术决策

- **协议选择**：Hysteria2（原生支持用户级流量统计）
- **架构原则**：第一性原理 + 奥卡姆剃刀
- **实施策略**：技术验证优先，渐进式开发

## 快速开始

### 理解核心架构

```
用户客户端
    ↓
TUN 虚拟网卡（不变）
    ↓
sing-box 核心（配置为 Hysteria2）
    ↓
Hysteria2 协议
    ↓
服务端节点集群
    ↓
Metering Service（流量汇总）
```

### 关键特性

1. **用户级流量统计**：Hysteria2 原生支持，通过 Traffic Stats API 获取
2. **多节点流量汇总**：所有节点流量统计汇总到统一 Metering Service
3. **预付费配额管理**：用户通过 x402 支付 USDC 购买流量配额
4. **中心化扣减**：Metering Service 中心化扣减配额，配额耗尽时阻断连接

### 技术约束

- ✅ 不修改 sing-box 源码
- ✅ 虚拟网卡层保持不变
- ✅ 客户端已封装 sing-box Go 代码
- ✅ 只需修改配置文件

## 术语表

| 术语 | 说明 |
|------|------|
| **流量配额** | 用户预付费购买的流量额度，永久有效直到用完 |
| **预付费购买** | 用户通过 x402 向服务商支付 USDC，兑换流量配额 |
| **流豆账户** | 用户自己持有助记词的钱包（Base 链上） |
| **供应商目录** | 来自链上注册合约的供应商列表 |
| **Metering Service** | 流量统计汇总和配额管理服务 |
| **Traffic Stats API** | Hysteria2 内置的用户级流量统计 API |
