## 3. 客户端模块

## 3.1 OpenMesh 商用目录层

职责：

- 读取链上供应商列表
- 加载 metadata
- 展示商用供应商卡片
- 触发切换

## 3.2 Wallet / Signer Session 层

职责：

- 创建 / 恢复钱包
- 解锁钱包
- 管理 signer session
- 自动签名 x402 支付

## 3.3 Receipt Store

职责：

- 保存最近支付收据
- 保存最近用量凭条
- 支持恢复和重放

## 3.4 Provider Switch Coordinator

职责：

- 请求供应商配置
- 安装供应商配置
- 管理当前商用供应商状态

---

## 4. 商用供应商参考服务端模块

## 4.1 Config Service

职责：

- 根据地址和当前状态生成接入配置
- 返回短期凭据

## 4.2 Metering Service

职责：

- 读取数据平面统计
- 维护地址 / session 的桶剩余量
- 在低于阈值时触发续费请求

## 4.3 x402 Payment Verifier

职责：

- 校验客户端自动支付请求
- 在支付通过后向 Metering Service 通知放行下一桶

## 4.4 Receipt Service

职责：

- 生成 PaymentReceipt
- 生成 UsageReceipt
- 提供凭条查询和恢复接口

## 4.6 Provider Withdrawal Service

职责：

- 在供应商请求提现时执行平台手续费逻辑
- 对接供应商收款合约和平台分成逻辑

注意：

- 该服务不进入用户支付热路径

## 4.5 sing-box Adapter

职责：

- 把地址或地址派生值映射成数据平面用户
- 下发、更新、停用用户配置
- 汇总节点统计

---

## 5. 数据平面协议建议

V1 参考实现建议优先选择：

- `Shadowsocks multi-user + managed`

原因：

- 官方多用户能力明确
- 管理 API 明确
- 最接近当前现有客户端路径

但协议层必须保持开放，未来可支持：

- `Trojan`
- `Hysteria2`
- `VLESS`

前提是它们满足：

- 地址 / session 级身份
- 流量统计
- 动态停供

## 6. 热路径设计

### 6.1 免费桶

供应商向新用户开放首个免费桶，例如 40MB。

### 6.2 预续费阈值

当剩余桶流量低于阈值时，由 Metering Service 发起下一桶续费，不等待完全耗尽。

默认建议：

- 流量桶大小：40MB
- 预续费阈值：5MB 或 20%

### 6.3 自动支付

客户端在 signer session 内自动签名 x402 支付。  
若超出单次或每日限额，则回退为人工确认。

### 6.4 放行

支付通过后，供应商立即放行下一桶。  
链上轮询只作为异步对账，不作为热路径门禁。

### 6.5 凭条同步

在支付放行和会话结束两个时机，供应商都必须把最新有效凭条返回给客户端。  
客户端应立即本地持久化，以便后续恢复和 token 结算。

## 7. MeshFluxMac 实现落点

本仓库当前只直接改造 `MeshFluxMac`。  
因此客户端内至少要新增：

- 商用供应商 registry 读取模块
- 钱包 signer session 管理模块
- 商用供应商配置请求与安装协调器
- 收据与凭条本地存储模块
- 商用供应商状态视图

---

## 8. 关键结论

1. 官方临时受限访问继续走简单单节点方案
2. 商用供应商必须引入独立的 provider 参考服务端
3. `sing-box` 是数据平面，不是账本
4. 热路径核心是“预续费 + 自动签名 + 立即放行”
