## 验收目标

验证流量采集器升级是否满足生产环境要求,包括并发采集、重试机制、节点故障隔离等功能。

## 功能测试

### 测试 1: 并发采集多节点

**测试步骤**:
```bash
# 启动多个测试节点
./scripts/start_test_nodes.sh 5

# 运行采集器
go test -v internal/collector -run TestCollectAll
```

**预期结果**:
- [ ] 可以并发采集 5 个节点
- [ ] 采集时间 < 1 秒
- [ ] 流量数据正确汇总

### 测试 2: 重试机制

**测试步骤**:
```bash
# 模拟节点间歇性故障
go test -v internal/collector -run TestRetryMechanism
```

**预期结果**:
- [ ] 失败后自动重试
- [ ] 使用指数退避
- [ ] 最多重试 3 次

### 测试 3: 节点故障隔离

**测试步骤**:
```bash
# 模拟节点持续故障
go test -v internal/collector -run TestNodeIsolation
```

**预期结果**:
- [ ] 连续失败 3 次后隔离节点
- [ ] 隔离期间不再采集该节点
- [ ] 节点恢复后自动重新启用

### 测试 4: 结构化日志

**测试步骤**:
```bash
# 运行采集器并检查日志
go run cmd/metering/main.go 2>&1 | jq .
```

**预期结果**:
- [ ] 日志为 JSON 格式
- [ ] 包含关键字段(node_id, user_id, tx, rx)
- [ ] 日志级别正确

## 性能测试

### 测试 5: 采集性能

**预期结果**:
- [ ] 采集 5 个节点 < 1 秒
- [ ] 采集成功率 > 99%
- [ ] 内存使用合理

## 验收标准

### 通过条件
- [ ] 所有功能测试通过
- [ ] 性能达标
- [ ] 日志完整清晰

### 失败条件
- [ ] 任何功能测试失败
- [ ] 性能不达标
- [ ] 节点故障影响其他节点
