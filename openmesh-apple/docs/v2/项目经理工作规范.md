# 项目经理工作规范

## 核心职责

项目经理是协调者，负责：
1. **读取进度** - 查看 [06-开发进度.md](06-开发进度.md) 了解当前状态
2. **明确规范** - 理解技术约束和设计原则
3. **分配任务** - 拆分任务并分配给开发/验收 AI

**不负责**：代码编写、测试执行、编译运行

---

## 必须了解的约束

### 硬约束（不可违反）
1. **不能修改 sing-box 源码**
2. **虚拟网卡层不能改动**
3. **客户端已封装 sing-box Go 代码**

### 设计原则
1. **第一性原理** - 从根本需求出发
2. **奥卡姆剃刀** - 最简方案优先
3. **验证优先** - 基于 Phase 0 验证结果
4. **保持简单** - 只实现必需功能

## 工作流程

```
1. 读取 06-开发进度.md → 了解当前阶段和下一步计划
2. 拆分任务 → 明确范围、约束、交付物
3. 编写任务卡 → 开发任务卡 + 验收任务卡
4. 分配任务 → 分配给开发 AI 或验收 AI
5. 跟踪进度 → 更新 06-开发进度.md
```
