# V2 订阅系统集成测试指南

## 测试概述

本文档描述如何从头开始测试 V2 订阅系统的完整功能。V2 的核心改进是支持**一个钱包为多个 VPN 身份订阅服务**。

**测试环境**：
- 网络：Base Sepolia 测试网
- 合约地址：`0x16D6D1564942798720CB69a6814bc2C53ECe23a1`
- USDC 地址：`0x036CbD53842c5426634e7929541eC2318f3dCF7e`
- 测试套餐：Plan 3 (0.1 USDC / 30 分钟)

## 测试前准备

### 1. 环境检查

确认以下配置正确：

```bash
# 检查后端配置
cd docs/V2_design/validation/phase4
cat .env | grep VPN_SUBSCRIPTION_CONTRACT
# 应该显示: VPN_SUBSCRIPTION_CONTRACT=0x16D6D1564942798720CB69a6814bc2C53ECe23a1

cat frontend/app.js | grep CONTRACT_ADDRESS
# 应该显示: CONTRACT_ADDRESS: '0x16D6D1564942798720CB69a6814bc2C53ECe23a1'
```

### 2. 准备测试钱包

需要准备：
- **1 个付款钱包**（MetaMask）：需要有 USDC 余额（至少 0.3 USDC）
- **3 个 VPN 身份地址**：可以是任意地址（不需要有余额）

```bash
# 生成 3 个测试身份地址（示例）
# 身份 1: 0x1111111111111111111111111111111111111111
# 身份 2: 0x2222222222222222222222222222222222222222
# 身份 3: 0x3333333333333333333333333333333333333333
```

### 3. 获取测试 USDC

在 Base Sepolia 测试网获取 USDC：
1. 访问 Base Sepolia Faucet 获取测试 ETH
2. 使用 USDC Faucet 或 DEX 获取测试 USDC
3. 确保钱包至少有 0.3 USDC（用于 3 个订阅）

## 常见问题排查

### 问题 1：订阅失败 - "identity already bound"

**原因**：该 VPN 身份已经有活跃订阅

**解决**：
1. 检查该身份是否已被其他钱包订阅
2. 如果是自己的订阅，先取消或等待到期
3. 使用不同的 VPN 身份地址

### 问题 2：签名失败

**原因**：EIP-712 Domain 或 Types 不匹配

**解决**：
1. 确认前端和后端的 Domain version 都是 '2'
2. 确认 CANCEL_INTENT_TYPES 包含 identityAddress 字段
3. 检查合约地址是否正确

### 问题 3：自动续费不工作

**原因**：续费服务未启动或配置错误

**解决**：
1. 检查后端日志，确认续费服务已启动
2. 检查 USDC 余额和授权额度
3. 检查订阅的 autoRenewEnabled 状态

### 问题 4：Paymaster 拒绝赞助

**原因**：合约地址未添加到白名单

**解决**：
1. 登录 CDP Dashboard
2. 检查 Paymaster 白名单配置
3. 确认包含所有 V2 函数签名
