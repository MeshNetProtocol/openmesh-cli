# 模块 2: 区块链支付集成模块

## 模块状态

**当前状态**: ⏳ 待开发  
**最后更新**: 2026-04-06

## 1. 模块概述

本模块负责处理用户的区块链支付,验证交易有效性,并在支付成功后创建订阅记录。支持自动续费机制。

### 1.1 核心功能

- **支付接收**: 接收用户的区块链支付交易
- **交易验证**: 验证支付交易的真实性和有效性
- **订阅激活**: 支付成功后自动激活用户订阅
- **自动续费**: 订阅到期前自动从用户钱包扣款
- **支付回调**: 提供支付回调接口供区块链网络调用
- **退款处理**: 支持订阅取消后的退款流程

## 2. 技术方案

### 2.1 核心技术选型

| 技术组件 | 选型 | 用途 |
|---------|------|------|
| 区块链网络 | Ethereum/Polygon | 支付网络 |
| 智能合约 | Solidity | 订阅支付合约 |
| Web3 库 | ethers.js | 与区块链交互 |
| 事件监听 | Go + ethclient | 监听支付事件 |

### 2.2 智能合约设计

**功能**:
- 接收用户支付
- 记录订阅信息(用户地址、订阅时长、金额)
- 触发支付事件,通知后端服务
- 支持自动续费授权

### 2.3 支付验证流程

```
1. 用户发起支付交易 → 智能合约
2. 智能合约触发 PaymentReceived 事件
3. 后端监听事件 → 获取交易哈希
4. 验证交易: 检查金额、发送者、接收者
5. 验证通过 → 激活订阅
```
