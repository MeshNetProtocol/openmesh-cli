## 1. 模块概述

本模块负责管理用户订阅的完整生命周期,包括创建、激活、续费、暂停和取消订阅,并协调与 Xray 服务器的用户状态同步。

### 1.1 核心功能

- **订阅生命周期管理**: 创建、激活、续费、暂停、取消订阅
- **用户状态同步**: 将订阅状态同步到 Xray 服务器(添加/删除用户)
- **定时任务调度**: 自动检查到期订阅并触发续费或停服
- **订阅查询**: 提供 API 查询用户订阅状态和历史
- **多服务器协调**: 管理用户在多个 Xray 服务器的访问权限
- **通知服务**: 发送订阅到期、续费成功/失败等通知

## 2. 技术方案

### 2.1 核心技术选型

| 技术组件 | 选型 | 用途 |
|---------|------|------|
| Web 框架 | Go + Gin | HTTP API 服务 |
| 数据库 | PostgreSQL | 订阅数据存储 |
| 缓存 | Redis | 用户状态缓存 |
| 定时任务 | Go cron | 任务调度 |

### 2.2 订阅状态机

**状态定义**:
- `pending`: 等待支付确认
- `active`: 订阅有效,用户可访问
- `expiring`: 即将到期(24 小时内)
- `expired`: 订阅已过期,用户无法访问
- `suspended`: 订阅暂停(如违规)
- `cancelled`: 订阅已取消

**状态转换**:
```
pending → (支付成功) → active
active → (接近到期) → expiring
expiring → (续费成功) → active
expiring → (续费失败) → expired
active → (手动暂停) → suspended
suspended → (恢复) → active
```
