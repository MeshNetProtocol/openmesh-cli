## 1. 需求描述

### 1.1 业务需求

作为 VPN 订阅服务的核心功能，需要实时统计和记录每个用户在所有 Xray 服务器上的流量使用情况，用于：

1. **计费依据**: 按流量计费的订阅套餐需要准确的流量统计
2. **配额管理**: 限制用户的流量使用，超过配额自动停服
3. **用户透明**: 用户可以实时查看自己的流量使用情况
4. **多服务器支持**: 用户可以连接到不同地区的多个服务器，需要汇总所有服务器的流量
5. **数据持久化**: 防止服务器重启后流量数据丢失

### 1.2 功能需求

#### FR1: 用户级别流量统计
- **描述**: 实时记录每个用户的上行和下行流量
- **优先级**: P0（必须）
- **验收标准**: 
  - 能够查询到每个用户的实时流量数据
  - 流量统计精度误差 < 1%

#### FR2: 多服务器流量汇总
- **描述**: 用户可以连接到多个 Xray 服务器，需要汇总所有服务器的流量
- **优先级**: P0（必须）
- **验收标准**:
  - 支持配置多个 Xray 服务器
  - 能够查询每个服务器的独立流量
  - 能够计算用户的总流量（所有服务器之和）

#### FR3: 流量持久化
- **描述**: 将流量数据持久化到文件，防止 Xray 重启后数据丢失
- **优先级**: P0（必须）
- **验收标准**:
  - Xray 重启后，历史流量数据不丢失
  - 能够自动检测 Xray 重启并保留历史数据
  - 流量数据每 5 秒自动保存

#### FR4: WebSocket 实时推送
- **描述**: 通过 WebSocket 实时推送流量更新到 Web 客户端
- **优先级**: P1（重要）
- **验收标准**:
  - Web 客户端连接 WebSocket 后能实时接收流量更新
  - 支持断线重连
  - 延迟 < 5 秒

#### FR5: Web 界面展示
- **描述**: 在 Web 界面上展示每个用户在每个服务器的流量和总流量
- **优先级**: P0（必须）
- **验收标准**:
  - 显示每个服务器的上行/下行流量
  - 显示用户的总流量
  - 流量自动格式化为人类可读格式（B/KB/MB/GB）

### 1.3 非功能需求

#### NFR1: 性能要求
- 流量查询延迟 < 100ms
- 支持至少 1000 个并发用户
- 流量统计更新间隔 5 秒

#### NFR2: 可靠性要求
- 流量数据不丢失
- 服务可用性 > 99.9%
- 自动故障恢复

#### NFR3: 可扩展性要求
- 支持动态添加/删除 Xray 服务器
- 支持水平扩展

## 2. 技术方案

### 2.1 核心技术选型

| 技术组件 | 选型 | 理由 |
|---------|------|------|
| VPN 服务器 | Xray-core | 高性能、支持 Stats API |
| 后端语言 | Go | 高性能、并发支持好 |
| 流量查询 | Xray Stats API | 官方 API，稳定可靠 |
| 数据持久化 | JSON 文件 | 简单、易于调试 |
| 实时推送 | WebSocket | 实时性好、双向通信 |
| 前端 | HTML/JS | 轻量、易于开发 |

### 2.2 Xray 流量统计配置

Xray 需要启用用户级别的流量统计：

```json
{
  "api": {
    "tag": "api",
    "services": ["HandlerService", "StatsService"]
  },
  "stats": {},
  "policy": {
    "levels": {
      "0": {
        "statsUserUplink": true,
        "statsUserDownlink": true
      }
    },
    "system": {
      "statsUserUplink": true,
      "statsUserDownlink": true
    }
  }
}
```

### 2.3 流量持久化机制

**核心思路**: 定期查询 Xray 流量，计算增量并累加到历史总流量

**实现原理**:
```
1. 定时任务每 5 秒查询所有 Xray 服务器的流量
2. 对每个用户的每个服务器:
   - 如果当前流量 < 上次流量 → Xray 重启了
     - 将上次流量累加到历史总流量
     - 重置上次流量为当前值
   - 否则 → 正常情况
     - 计算增量 = 当前流量 - 上次流量
     - 累加到历史总流量
     - 更新上次流量为当前值
3. 汇总所有服务器的流量
4. 保存到 users.json 文件
5. 触发 WebSocket 推送事件
```

**数据结构**:
```json
{
  "xray_servers": [
    {
      "name": "Server-1",
      "address": "127.0.0.1:10085",
      "location": "US-West"
    }
  ],
  "users": [
    {
      "email": "0x1234...",
      "uuid": "11111111-1111-1111-1111-111111111111",
      "enabled": true,
      "servers": {
        "Server-1": {
          "total_uplink": 1024000,
          "total_downlink": 5120000,
          "last_xray_uplink": 512000,
          "last_xray_downlink": 2560000
        }
      },
      "total_uplink": 1024000,
      "total_downlink": 5120000
    }
  ]
}
```

### 3.2 核心组件

#### 3.2.1 流量统计模块 (Go)

**文件**: `auth-service/main.go`

**核心函数**:
- `queryUserTrafficFromServer()`: 查询单个服务器的用户流量
- `UpdateTrafficStats()`: 更新所有用户的流量统计
- `Save()`: 持久化到 users.json

**定时任务**:
```go
go func() {
    ticker := time.NewTicker(5 * time.Second)
    defer ticker.Stop()
    for range ticker.C {
        if err := config.UpdateTrafficStats(); err != nil {
            log.Printf("Failed to update traffic stats: %v", err)
        }
    }
}()
```

#### 3.2.2 WebSocket 服务器 (Go)

**端点**: `ws://localhost:8080/ws`

**功能**:
- 维护所有活跃的 WebSocket 连接
- 监听流量更新事件
- 广播到所有连接的客户端

#### 3.2.3 Web 前端 (HTML/JavaScript)

**功能**:
- 连接 WebSocket
- 接收流量更新
- 动态更新界面显示
