## 1. 模块概述

本模块负责实时统计用户在多个 Xray 服务器的流量使用情况,支持流量持久化和 WebSocket 实时推送。这是整个订阅式 VPN 系统的核心基础模块。

### 1.1 核心功能

- **用户级别流量统计**: 实时记录每个用户的上行和下行流量
- **多服务器支持**: 同时统计多个 Xray 服务器的流量
- **流量汇总**: 自动汇总用户在所有服务器的总流量
- **流量持久化**: 定期保存流量数据到文件,防止 Xray 重启后数据丢失
- **重启检测**: 自动检测 Xray 重启,保留历史流量数据
- **实时推送**: 通过 WebSocket 向客户端实时推送流量更新
- **动态用户管理**: 支持动态添加/删除用户,无需重启 Xray

## 2. 技术方案

### 2.1 核心技术选型

| 技术组件 | 选型 | 用途 |
|---------|------|------|
| 流量统计 API | Xray Stats API | 获取用户级别流量数据 |
| 用户管理库 | xtlsapi (Python) | 动态添加/删除用户 |
| HTTP 服务 | Go + Gin | 订阅管理和流量统计服务 |
| 实时推送 | Gorilla WebSocket | 实时推送流量更新 |
| 数据持久化 | JSON 文件 | 保存用户配置和流量数据 |

### 2.2 流量持久化机制

**问题**: Xray 的流量统计存储在内存中,重启后数据丢失

**解决方案**: 定期查询并持久化流量数据

**实现原理**:
```
1. 定时任务每 5 秒查询所有 Xray 服务器的流量
2. 计算增量: 当前流量 - 上次记录的流量
3. 检测重启: 如果当前流量 < 上次流量,说明 Xray 重启了
4. 累加到历史总流量并保存到 users.json
```
