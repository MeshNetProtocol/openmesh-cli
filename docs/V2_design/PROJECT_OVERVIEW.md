# V1 版本

**目标**: 支持 1000 个并发客户端，单实例部署，MVP 功能  
**状态**: 规划已明确，进入开发  
**预计发布**: TBD

## V1 需求描述

### 业务需求

构建一个基于区块链地址认证的订阅式 VPN 服务系统，用户通过区块链支付订阅费用后，可以使用客户端访问 Xray 服务器进行 VPN 转发。

**核心目标**:
- **用户规模**: 支持 1000 个并发客户端
- **去中心化身份**: 使用区块链地址作为用户唯一标识
- **订阅制付费**: 支付后自动获得访问权限，支持自动续费
- **基础流量统计**: 记录用户流量使用情况
- **简化架构**: 单实例部署，2-3 台 Xray 服务器

### 技术需求

- **可靠性**: 订阅状态变更可靠执行，失败自动重试
- **安全性**: 通信加密，防止支付欺诈
- **性能**: API 响应时间 < 200ms，支持 1000 用户

### 技术栈

**后端**:
- **Go**: 目标订阅管理服务、Xray gRPC 客户端
- **PostgreSQL**: 目标订阅记录、支付记录、流量数据存储
- **Xray**: VPN 流量转发服务器

**客户端** (已有实现):
- **macOS/iOS**: Swift (openmesh-apple)
- **Windows**: C#/.NET (openmesh-win)
- **Android**: Kotlin/Java (openmesh-android)

**区块链**:
- **Base Mainnet**: V1 目标生产网络
- **USDC**: 支付代币
- **CDP**: Smart Account + Spend Permissions

## V1 模块功能

### 模块 1: 智能合约

**状态**: 方案已收敛，开发进行中

**功能**:
1. **订阅绑定与授权**: 绑定 identity 与 payer，处理 ERC-2612 Permit 签名授权
2. **自动扣费**: 使用 Spend Permissions 执行续费扣费
3. **扣费去重**: 通过唯一 chargeId 防止重复扣费
4. **链上最小职责**: 仅处理授权与扣费，不在链上管理套餐和订阅状态

**技术方案**:
- VPNCreditVault 合约
- 服务端管理套餐配置与订阅状态
- Foundry 测试框架
- Base Mainnet 部署

### 模块 2: 订阅管理服务

**状态**: 已完成核心实现

**功能**:
1. **订阅生命周期**: 创建、激活、续费、取消订阅
2. **用户状态同步准备**: 为 Xray 同步提供订阅状态来源
3. **定时任务**: 自动续费检查、过期清理基础结构
4. **订阅查询 API**: 查询用户订阅状态

**技术方案**:
- Go + Gin 框架
- PostgreSQL 数据库
- 订阅状态机: pending → active → expired/renewed
- 单实例部署

### 模块 3: Xray 集成模块

**状态**: 已完成核心实现

**功能**:
1. **用户管理**: 已支持根据订阅状态添加/删除 Xray 用户
2. **基础流量统计**: 已支持定期查询用户流量
3. **流量持久化**: 已保存流量数据到 PostgreSQL
4. **后台展示**: 已在管理后台展示 uplink / downlink
5. **后续能力**: 流量配额检查仍待补充

**技术方案**:
- Go Xray 客户端封装
- 用户同步 + 流量统计后台服务
- 当前以单实例后端 + 单 Xray 联调为主，后续扩展到 2-3 台服务器

### 模块 4: 客户端

**状态**: 基础能力已有，目标实现需适配

**代码仓库**:
- macOS/iOS: openmesh-apple
- Windows: openmesh-win
- Android: openmesh-android

**适配工作**:
1. **API 对接**: 对接订阅管理服务 API
2. **Xray 配置**: 适配 Xray 服务器配置格式
3. **功能测试**: 测试订阅购买和 VPN 连接

# V2 版本

**目标**: 支持 10000+ 并发客户端，多实例部署，高级功能  
**状态**: 规划阶段  
**预计启动**: V1 发布后

## V2 需求描述

### 业务需求

在 V1 基础上扩展，支持更大规模用户和更多高级功能。

**核心目标**:
- **用户规模**: 支持 10000+ 并发客户端
- **高可用性**: 多实例部署，支持故障转移
- **实时性**: WebSocket 实时流量推送
- **多服务器**: 支持 10+ Xray 服务器，流量汇总
- **高级功能**: 通知服务、高级监控告警

### 技术需求

- **高可用性**: 服务器故障不影响其他服务器，支持故障转移
- **实时性**: 用户状态变更实时生效，流量统计实时更新
- **可扩展性**: 支持水平扩展，可添加更多服务器
- **性能**: API 响应时间 < 100ms，支持 10000+ 用户
