## 一、产品定位

OpenMesh V2 是流量行业的 OpenRouter。

OpenRouter 让用户能够统一接入不同 AI 模型供应商，OpenMesh 让用户能够统一接入不同 VPN 流量供应商。两者的核心逻辑一致：**平台负责统一接入和结算协议，内容（流量）由供应商提供，用户自由选择**。

对应关系如下：

| OpenRouter | OpenMesh V2 |
|-----------|-------------|
| AI 模型供应商 | 流量供应商 |
| 模型套餐（价格/月度 Token 额度） | 订阅套餐（价格/月度流量额度） |
| API Key | x402 订阅凭证 |
| 统一 API 接入 | 统一 VPN 接入（sing-box） |
| 按模型订阅计费 | 按供应商订阅计费 |

**计费模式**：月度订阅制。用户向供应商支付固定月费（USDC），通过 x402 协议完成去中心化结算，到期自动续订。

## 二、平台角色定义

### 2.1 三类角色

| 角色 | 描述 |
|------|------|
| 用户 | 购买订阅、使用 VPN 服务的终端用户 |
| 流量供应商 | 提供 VPN 节点和流量、自定义订阅套餐、独立管理用户订阅的服务提供方 |
| OpenMesh 平台 | 提供统一接入协议（x402 订阅）、流量转发规范、供应商发现机制，以及官方参考实现 |

### 2.2 平台职责边界

OpenMesh 平台的核心职责是：

- **流量转发**：提供统一的 VPN 数据通道规范（基于 Hysteria2），确保任何经过认证的用户流量都能正确转发
- **x402 订阅协议**：提供标准化的订阅支付和验证规范，供应商必须遵守这个协议
- **供应商发现**：维护去中心化的供应商注册表（链上），让用户能够发现并选择供应商
- **官方参考实现**：提供一套完整的、开源的流量管理和套餐管理系统，供无自建能力的供应商直接使用

**平台不做的事**：
- 不托管用户资金
- 不参与供应商与用户之间的支付结算过程
- 不强制供应商使用官方管理系统——只要遵守 x402 订阅协议和流量转发规范，供应商可以使用自己的后端系统

## 三、供应商接入模式

### 3.1 两种接入模式

**模式 A：使用官方系统（推荐新供应商）**

供应商没有自建系统，直接部署 OpenMesh 官方开源的流量管理和套餐管理系统。供应商只需要配置自己的节点和套餐，其余功能开箱即用。

**模式 B：使用自有系统**

供应商已有成熟的流量管理/用户管理系统，不需要使用官方实现。只要其系统实现了 x402 订阅协议接口和标准的流量转发规范，即可接入 OpenMesh 平台被用户发现和使用。

### 3.2 官方系统（本期需要实现）

官方系统是 OpenMesh 平台本期交付的核心服务端工程，包含两个子系统：

**流量管理系统**

负责识别每个用户的流量，并根据其订阅等级执行对应的服务策略（月度流量额度监控与 Fair Use 执行）。

**套餐管理系统**

负责供应商定义和管理订阅套餐，处理用户的订阅购买、续订、状态查询和取消。

两个子系统均以开源形式发布，供应商可以自行部署和定制，或直接使用 OpenMesh 托管版本。

## 四、订阅套餐设计

### 4.1 套餐由供应商定义和控制

每个流量供应商完全控制自己提供的订阅套餐，包括等级划分、价格设定、各等级的服务参数。OpenMesh 平台不参与套餐内容的定义，只规范套餐的格式和发布方式。

### 4.2 套餐等级（Tier）

订阅套餐按等级划分，不同等级在以下维度上有明确区别：

| 维度 | 说明 |
|------|------|
| **月费价格** | 以 USDC 计，每 30 天收取一次 |
| **月度流量额度（Fair Use）** | 当月流量参考上限，类似大模型的月度 Token 额度；超出后供应商可选择降速，不得直接断线 |
| **节点覆盖范围** | 该等级可使用哪些地区的节点 |
| **套餐名称** | 用户可识别的名称，如"基础版"、"标准版"、"旗舰版" |

**示例（供应商自行定义，以下仅为示意）**：

| 等级 | 月费 | 月度流量额度 |
|------|------|-------------|
| 基础版 | 3 USDC | 100 GB |
| 标准版 | 6 USDC | 300 GB |
| 旗舰版 | 12 USDC | 1 TB |

**说明**：
- 用户可在任意数量的设备上使用，不限制设备数
- 月度流量额度为 Fair Use 参考值，超出后供应商可降速，不得直接断开用户连接
- 不区分上行和下行，按总流量计算
- 订阅续订时，流量使用量自动重置为 0

（以上数字均为示例，实际由各供应商自主设定）

### 4.3 套餐管理规则

- 供应商可以定义任意数量的等级套餐，但至少需要上架一个有效套餐，才会在用户端展示
- 套餐可以随时上架或下架，下架不影响已订阅该套餐用户的当前周期服务
- 同一供应商的套餐等级，名称必须对用户清晰可辨，不允许模糊命名
- 套餐价格一旦有用户订阅，在该用户当前订阅周期内不得变更

## 五、功能需求

### 5.1 用户订阅购买

用户通过客户端浏览供应商和套餐，完成订阅购买，流程不超过 3 步。

**用户操作流程**

1. 浏览供应商列表，查看各供应商的套餐等级、月费、月度流量额度（Fair Use）、节点覆盖
2. 选择供应商和具体套餐等级，查看费用明细
3. 解锁钱包，通过 x402 协议完成支付
4. 订阅立即生效，VPN 服务立即可用，可在任意数量的设备上使用

**业务规则**

- 首次购买需用户主动解锁钱包并签名，系统不得自动发起首次扣费
- 支付成功后，VPN 服务必须在 1 分钟内可用
- 同一用户向同一供应商，在同一有效周期内不允许重复购买同一等级套餐
- 用户可以向多个不同供应商分别订阅，各订阅相互独立
- 不限制每个订阅的使用设备数量

### 5.2 订阅等级与流量管理（Fair Use）

**月度流量额度说明**

- 用户可在任意数量的设备上使用 VPN，不限制设备数
- 月度流量额度为 Fair Use 参考值，按当月上行+下行流量总和计算
- 流量达到套餐额度 80% 时，客户端发出提醒通知
- 流量超出套餐额度后，供应商可选择降速或继续提供服务，**不得直接断开用户连接**
- 订阅续订时，流量使用量自动重置为 0

**用量展示**

- 客户端提供用量展示界面，清晰呈现：
  - 套餐月度流量额度
  - 当月已使用流量
  - 剩余流量及使用百分比
  - 本周期流量重置日期
- 界面设计参考大模型（如 OpenRouter、Claude）的用量余额显示风格
- 数据延迟不超过 1 分钟

### 5.3 订阅生命周期

系统需要准确维护每个订阅的状态，并依据状态决定是否允许用户建立 VPN 连接。

**订阅状态及其含义**

| 状态 | 含义 | 用户是否可以使用 VPN |
|------|------|---------------------|
| 有效（Active） | 订阅在有效期内 | 可以 |
| 宽限期（Grace） | 到期后 24 小时内 | 可以 |
| 已过期（Expired） | 超出宽限期、续订未成功 | 不可以，连接被拒绝 |
| 已取消（Canceled） | 用户主动取消，等当前周期结束 | 当期内可以 |

**业务规则**

- 订阅有效期为 30 天，从支付成功时刻起算
- 有效期结束后进入 24 小时宽限期，宽限期内服务继续
- 宽限期结束后仍未续订，系统立即拒绝该用户的 VPN 连接请求
- 宽限期内续订成功，新周期从原到期时间起算，不惩罚延迟续订的用户
- 用户取消订阅后，当前付费周期内服务不中断，到期时自然停止

### 5.4 自动续订

**触发时机**

- 订阅到期前 3 天，系统开始尝试自动续订
- 首次尝试失败后，宽限期内每 6 小时自动重试一次，直到成功或宽限期结束

**执行方式**

- 自动续订由客户端发起，利用用户已授权的 Signer Session 构建续订支付并提交
- 服务端验证签名并更新订阅状态，不持有用户私钥，不从服务端主动发起扣款
- 续订成功后套餐等级保持不变（与上一周期相同）；用户若希望续订时升级，需手动操作

**用户对自动续订的控制权**

- 用户可以随时在客户端开启或关闭自动续订
- 用户可以设置自动续订的月度总金额上限
- Signer Session 的自动签名仅限订阅续订场景，单次金额不超过套餐当期价格

**续订失败处理**

- 续订失败后立即推送通知，明确告知失败原因（余额不足 / 需要重新授权）
- 进入宽限期，客户端持续显示"请尽快续订"提示
- 宽限期结束仍未续订，服务暂停

### 5.5 套餐升级与降级

用户可以在订阅期内更换同一供应商的不同等级套餐。

- **升级（换更高等级）**：需补交当前周期剩余天数的差价，立即生效，流量额度立即更新
- **降级（换更低等级）**：当前周期维持原等级服务，下个续订周期按新等级计费
- 升级/降级不重置订阅有效期

### 5.6 订阅管理

**用户可以查看的信息**

- 所有活跃订阅的供应商名称、套餐名称、套餐等级、到期时间、剩余天数
- 用量信息（参考大模型用量显示风格）：
  - 套餐月度流量额度
  - 当月已使用流量
  - 剩余流量及使用百分比
  - 本周期流量重置日期
- 自动续订开启状态
- 历史支付记录（时间、金额、套餐等级、覆盖周期）

**用户可以执行的操作**

- 开启或关闭自动续订
- 手动发起续订
- 升级套餐等级（立即生效）
- 取消订阅

### 5.7 服务暂停与恢复

**暂停**

- 宽限期截止后，该用户的 VPN 连接被断开，新的连接请求被拒绝
- 客户端明确提示"订阅已过期，请续订后使用"，不允许显示模糊的网络错误

**恢复**

- 续订成功后，VPN 服务在 1 分钟内自动恢复，用户无需手动操作

### 5.8 流量监控（Fair Use）

**用户侧**

- 客户端实时展示当月已使用流量（延迟不超过 1 分钟）
- 流量达到套餐额度 80% 时，客户端推送提醒通知
- 流量超出套餐额度后，客户端给出提示，说明当前供应商的 Fair Use 策略（降速或继续）

**供应商侧**

- 供应商可通过管理界面查看各订阅等级、各用户的流量使用概况
- 供应商可以为每个套餐等级分别设置月度流量额度和超额处理策略

**重要约束**

- 流量超额不得直接断开用户连接，服务中断的唯一条件是订阅过期
- 流量统计需满足 Fair Use 管理的基本要求

## 六、用户场景

### 场景 1：新用户首次订阅

张三第一次使用 OpenMesh。

1. 打开客户端，创建或导入钱包
2. 进入供应商列表，看到多家供应商，每家都展示了不同等级套餐的月费、月度流量额度（Fair Use）、节点覆盖
3. 他选择了一家供应商的"标准版"套餐（6 USDC/月，300 GB Fair Use 额度）
4. 查看费用明细后确认，Face ID 解锁钱包，支付完成
5. 客户端提示"订阅成功，有效至 2026-04-26，月度流量额度 300 GB"
6. 立即可以使用 VPN，可在任意设备上连接

---

### 场景 2：流量使用达到 80%，收到提醒

李四订阅了"基础版"（100 GB Fair Use 额度/月）。

1. 他在多台设备上使用 VPN，本月已使用 82 GB
2. 客户端推送通知："本月流量已使用 82%（82/100 GB），请注意"
3. 他继续使用，流量超出 100 GB 后，供应商对其连接进行降速处理
4. 客户端提示："本月流量已超出套餐额度，当前处于降速状态，可升级套餐恢复正常速度"
5. 他选择升级到"标准版"（300 GB），补交当前周期剩余天数的差价，速度立即恢复

---

### 场景 3：到期前自动续订成功

王五的订阅还有 3 天到期，钱包余额充足，Signer Session 有效。

1. 系统在后台静默发起续订支付，成功
2. 客户端推送通知："订阅已自动续订，服务有效至 2026-05-26，套餐等级不变，流量已重置"
3. 王五全程无需操作，服务无中断

---

### 场景 4：自动续订失败，宽限期内手动补救

钱七钱包余额不足。

1. 到期前 3 天，自动续订失败
2. 客户端推送通知："续订失败，原因：钱包余额不足。服务宽限至 2026-04-29，请尽快续订"
3. 钱七充值后，在客户端点击"立即续订"，支付成功
4. 新周期从原到期时间 2026-04-26 起算，宽限期内服务未曾中断，流量使用量重置为 0

---

### 场景 5：宽限期结束，服务暂停后恢复

孙八宽限期结束前未续订。

1. 宽限期截止后，VPN 连接断开
2. 客户端显示"您的订阅已过期，请续订后继续使用"
3. 孙八完成续订，不超过 1 分钟，VPN 自动恢复，流量使用量重置为 0

---

### 场景 6：切换供应商（官方系统 vs 自有系统供应商）

周九当前使用的是一家自建了系统的成熟供应商，现在想试试一家规模较小、使用官方系统的新供应商。

1. 在供应商列表中，两种供应商显示方式一致，用户无感知供应商后端用的是哪套系统
2. 周九选择新供应商的套餐，完成订阅购买
3. 客户端切换到新供应商节点，连接秒级恢复
4. 客户端提示"你还有一个旧供应商的订阅在有效期内，是否取消？"，周九自主决定保留或取消

---

### 场景 7：供应商使用官方系统上架套餐

一家新的流量供应商想接入 OpenMesh 平台，没有自建技术系统。

1. 供应商部署 OpenMesh 官方开源的流量管理系统和套餐管理系统
2. 在套餐管理系统中定义三个等级套餐（基础/标准/旗舰），设定各等级的价格、月度流量额度
3. 在链上完成供应商注册
4. 用户端即可发现该供应商并订阅其套餐，支付通过 x402 协议直接结算给该供应商

---

### 场景 8：供应商使用自有系统接入

一家已有成熟用户系统的供应商希望接入 OpenMesh 平台获取更多用户。

1. 该供应商在自有系统中实现 OpenMesh 定义的 x402 订阅协议接口
2. 按照 OpenMesh 流量转发规范配置其 Hysteria2 节点
3. 在链上完成供应商注册，提交套餐信息
4. 用户端即可发现并订阅，该供应商的实际订阅管理、用户管理由其自有系统处理，OpenMesh 只负责流量转发和 x402 支付验证

## 八、验收标准

### 8.1 官方系统

- [ ] 官方流量管理系统和套餐管理系统以开源形式交付，附部署文档
- [ ] 供应商可以独立部署官方系统，无需依赖 OpenMesh 平台托管
- [ ] 已有自建系统的供应商，只要实现 x402 协议接口，即可接入 OpenMesh 平台
- [ ] 用户侧无法感知一个供应商使用的是官方系统还是自建系统，体验一致

### 8.2 套餐与等级

- [ ] 供应商可以定义至少 3 个不同等级的订阅套餐
- [ ] 每个套餐等级可以独立设置价格和月度流量额度
- [ ] 客户端正确展示各套餐等级的差异，用户可以清晰对比
- [ ] 套餐下架后，已订阅该套餐的用户当期服务不受影响

### 8.3 月度流量（Fair Use）

- [ ] 客户端正确展示当月已使用流量和套餐额度，延迟不超过 1 分钟
- [ ] 流量达到套餐额度 80% 时，客户端推送提醒通知
- [ ] 流量超出套餐额度后，供应商可执行降速策略，但不得直接断开用户连接
- [ ] 超额后客户端明确告知用户当前降速状态，并提示可升级套餐
- [ ] 订阅续订时，流量使用量自动重置为 0

### 8.4 订阅购买与续订

- [ ] 支付成功后，VPN 服务在 1 分钟内可用
- [ ] 订阅到期前 3 天内，钱包余额充足且授权有效时，系统自动完成续订，服务无中断
- [ ] 自动续订失败时，用户立即收到通知，说明明确的失败原因
- [ ] 宽限期内续订成功，新周期从原到期时间起算

### 8.5 服务暂停与恢复

- [ ] 宽限期结束后，VPN 连接断开，客户端显示"订阅已过期"，不显示模糊的网络错误
- [ ] 续订成功后，不超过 1 分钟 VPN 服务自动恢复
- [ ] 服务中断的唯一条件是订阅过期，流量超额不触发断线

## 十、风险提示

| 风险 | 说明 | 建议 |
|------|------|------|
| 供应商自建系统接入质量参差不齐 | 各供应商自有系统对 x402 协议的实现可能存在偏差，影响用户体验 | 提供官方 SDK 和测试工具，供应商接入前需通过一致性测试 |
| 用户对 Fair Use 策略不理解 | 用户可能误以为订阅制代表无限流量，超额降速后感到意外 | 订购前必须清晰展示月度流量额度，不以小字形式隐藏 |
| 钱包余额不足导致自动续订失败率高 | 用户可能不维持钱包余额 | 在到期前 7 天向余额不足用户推送余额提醒 |
| Signer Session 过期导致无法自动续订 | Session 有有效期 | 续订失败提示中明确指引用户重新授权钱包 |
| 链上支付确认延迟 | 区块链交易存在确认等待时间 | 收到支付签名后即可先激活订阅，链上确认作为事后审计 |
